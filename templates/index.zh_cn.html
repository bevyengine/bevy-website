数据驱动

所有引擎和游戏逻辑都使用定制的实体组件系统 Bevy ECS
  • 快：大规模并行和高速缓存友好，并且是最快的 ECS（基于某些基准测试）
  • 简：Component 都是 Rust 结构体, System 都是 Rust 函数
  • 强：查询、全局资源、本地资源、变更检测、无锁并行调度器

2D 渲染器

为游戏和应用程序渲染实时 2D 图形
  • 特性：精灵表、动态纹理图集、相机、纹理、材质
  • 可扩展：自定义着色器、材质、渲染管道
  • 通用核心：构建于 Bevy 的 Render Graph 之上

3D 渲染器

现代且灵活的 3D 渲染器
  • 特性：光照、阴影、相机、网格、纹理、材质、gltf 加载
  • 可扩展：自定义着色器、材质、渲染管道
  • 通用核心：构建于 Bevy 的 Render Graph 之上

Render Graph

使用 Graph 结构组成自定义渲染管道
  • 并行：Render Graphs 自动并行渲染
  • 模块化：使用 Render Graph 节点构建可组合且可重用的渲染逻辑
  • 后端无关性：不绑定于特定的图形 API

跨平台

支持所有主流平台
  • Windows, MacOS, Linux, Web, iOS, Android

Bevy UI

专门为 Bevy 构建的自定义 ECS 驱动的 UI 框架
  • 直接构建在 Bevy 的 ECS、渲染器和场景插件之上
  • 使用代码动态编写 UI，或是使用 Bevy Scene 格式声明式地编写 UI
  • 使用熟悉的 "flex box" 模型布局你的 UI

场景

使用 Bevy 的场景系统创建、保存和加载 ECS 世界
  • 加载：加载场景时保留 Entity ID（对保存游戏非常有用）
  • 实例化：实例化使用新的 Entity ID 创建场景的链接副本
  • 热重载：对场景文件的更改会自动应用于正在运行的应用程序

声音

加载音频文件并按需播放
  • 将音频文件加载为资产
  • 使用音频资源播放音频资产

热重载

无需重启应用或重新编译，即可获得有关更改的即时反馈
  • 资产变动会立即反映在运行中的 Bevy 应用程序中
  • 目前可以热加载场景、纹理和网格
  • 可整合任何资产类型

高效编译时间

游戏开发是一个迭代过程，你不能一直等待编译
  • 使用 Bevy，在 "快速编译 "配置下，编译时间预计为 0.8~3.0 秒。
  • 与其他流行的 Rust 游戏引擎相比，后者可能需要 30 秒以上才能编译一个换行符的插入！

免费且开源

由开发者社区制作并为其服务的引擎
  • 100% 免费，并将永远如此
  • 根据宽松的 MIT 或 Apache 2.0 许可开放源代码
  • 无合同
  • 无许可费
  • 不削减销售额
