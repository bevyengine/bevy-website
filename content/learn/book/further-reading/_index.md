Hungry for more knowledge?
Bevy has a wide range of different learning materials, targeted at different use cases and learning preferences.

## Documentation Sources

Everyone learns differently, so we offer a variety of complementary paths:

- [Quickstart](/learn/quickstart): Build a simple game through a hands-on tutorial designed for absolute beginners.
- [The Book](/learn/book): Learn the core concepts that make Bevy work, then explore advanced topics needed to ship a production game.
- [docs.rs](https://docs.rs/bevy/latest/bevy/): Versioned API documentation that explains every struct, method, and function. Module and crate docs also provide an overview of specific areas of the code.
- [Feature Examples](https://bevyengine.org/examples): Explore Bevy's features by browsing runnable snippets.
- [Game Examples](https://bevyengine.org/examples/#games): Explore larger project structure and genre-specific ideas through playable game stubs.

## How to Use These Together

If you want to jump right in:

1. Complete the Quickstart Guide, learning by doing.
2. Start a simple project of your own: arcade games are a great choice.
3. Borrow from the examples when you're trying to figure out how to do something.
4. Consult the Bevy Book to when you encounter a new concept you don't understand or get stuck on a problem.
5. Refer to docs.rs while using specific APIs.

If you're the sort who prefers to understand what's going on before beginning:

1. Use the Quick Start Guide to get Bevy set up, then bail.
2. Read the Bevy Book cover-to-cover to build a clear mental model of how Bevy works.
3. Modify an existing game example to test out the features you've learned about.
4. Borrow from the examples when you're trying to figure out how to do something.
5. Refer to docs.rs while using specific APIs.
